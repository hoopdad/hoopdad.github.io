I'll start with the provocative idea, then get into an example.

## The "surface" argument

Dashboards were a universal target when we were in our data era, but now 
we are in the AI Agent era. The goal of a dashboard is gathering related data 
that leads to action or insight. That still sounds like a good idea, doesn't
it?

Your agents don't need to look at dashboards, and we need them to do work,
not you. The surface has changed from browser to agentic workspace.

For an individual contributor (IC), dashboards create more work, not less. 
I have seen people with access to agent workspaces start to build out
dashboards. They look fantastic, better than the past of most PPT's 
or web pages created by people who don't design websites or presentations.

At first glance, that person has enough action-oriented information to take
over the world. but now that person has lots of new places to look to read, 
interpret, decide and act. But our agents can do that. 

For taking action on their work, the surface used by that IC should be 
the agent workspace, not a web browser. Otherwise, the human remains the
bottleneck in the workflow.

## Agent Capabilities for Workflow Improvement

Agents can gather data, and assemble it into data stores or data visualizations.
No dispute there.

1. Agents can **gather** data or, when its consistent and structured data, agents can
write code to gather data (saving tokens).
2. Agents can **interpret** data, too. This is the reasoning stage: given the IC's 
instructions and context, it understands what data means for the IC, what thresholds are good and which are bad. 
3. Agents can **decide**, again with rules and context given by the IC, which of its 
available actions are appropriate for the conditions.
4. Agents can **take action** within the bounds of the instructions and permissions.

## Our Use Case: Planning for our Day

As a knowledge worker, I have a love/hate relationship with my calendar. There
are so many opportunities to do things, so many pulls from non-priority work.
Focusing on the right things is the most important skill to proving value and
finding fulfillment in work.

I will show the transition from Dashboard First to Agent First.

## Build a dashboard

Here's a prompt I could use to build a dashboard to look at my calendar and
point out to me which items drive value for my employer and which are things
that match my goals.

> Review my calendar for the next week. For each meeting and task, evaluate:
> 1. Does this directly contribute to my Q1 metric for delivering value to customers for 70% of my time?
> 2. Is this a commitment I made to someone else that I need to honor?
> 3. Does this align with my goal to work on topics that advance my career?
> 
> Flag anything that's neither urgent nor aligned with these priorities.
> Create a visual dashboard showing meetings by value category: High Priority, 
> Committed, Developmental, and Low Priority. Highlight any conflicts or back-to-back sessions. 
> Show it day-by-day.

![Overview given by Dashboard generated by prompt](/assets/2026/calendar-dashboard.png)

Now I can look at this dashboard and see that I've got a bunch of development items on my calendar,
and a few low priority meetings. I might need time back to work on the bunch of high priority items,
so now I can go through the calendar, day by day, to review.

![One Day Review given by Dashboard generated by prompt](/assets/2026/calendar-dashboard-day.png)

Now I've reviewed one day, and need to make some decisions. What are the agendas of these? Will any
be recorded so I can watch on my own time? Are any required by my manager? Do I have enough time to
do all my high priority work? And, I don't really have the context for the whole week.

The dashboard was nice, but it is not the end of the story. There is a lot of work left for me to do.

## Build an Agent

This use case is handled nicely by the Chief of Staff Agent. There are detailed instructions on how
to set this up. It will take you maybe 30 minutes to an hour for a basic agent, running in Scout,
Claude, Copilot, Copilot with Cowork, GitHub Copilot, etc. The instructions take you through the basics.
The more you put into that work, and the more you incorporate continuous feedback to the agent workspace,
the more you will get out of it.

That page above has you schedule it, but if you are curious without wanting to set it up, the net is you
would be able to prompt something like this, and get some really well-informed recommendations that you
can accept, ask follow-up questions, or choose to reject. But what you aren't doing is creating a complex
matrix of info in your head and solving on a second-class problem. Calendar management isn't where my value at work is derived, so I'm saying this task is not creating value.

> Review my calendar, emails, and tasks for today. Based on my Chief of Staff [CHIEF OF STAFF NAME], my
> Soul Code values and my goals, what is the ONE strategic priority I must nail today? Flag any conflicts
> with my non-negotiables and tell me if I have enough white space to think strategically.

That's an exmaple and you would likely refine it over time. But you get the idea. You likely will see
value in using automation and scheduling capabilities of your agentic workspace after a few days of
running that and starting to understand that there are insights leading to first class value that you
were missing because it is a complex task! More pwoer to those who do this work for a living!

## Takeaway

If the IC who developed a dashboard had an Agent First mindset, they would discover that the amount of
time spent on instructing an agent to create a dashboard is the same as the amount of time instructing
their agent to create a new loop. Think how they would obsess over the details: why is that calendar
item classified like that, I want the formatting different on the detail page, I don't need such a
big KPI header.

Instead, we are putting our mind into telling agents what to do. We are managing agents in this way,
not getting them to do tasks. Use the analogy of a manager and employees: a good manager is delegating,
giving boundaries, understanding what is good quality work, demanding auditability where appropriate.
But letting the IC do the work the way that the IC knows best.

I encourage you to start with this Chief of Staff Agent, then following along as we build out an
Agentic Operating System, letting the agents do the work that lets you add the most value to
your organization, your home, your society.
