# Jekyll GPT-5.5 Draft Series

This directory contains an isolated Jekyll draft series generated from `blog.txt` for later model comparison.

## Sequence and positioning

1. **Coding Assistants, Part 1: Decisions, Capabilities, and Expertise**  
   Positions AI coding assistants as managed delivery capacity, not magic autocomplete. Establishes the core capabilities: code generation, tool use, flow design, instruction following, Red Team review, TDD, critics, and infrastructure as code.

2. **Coding Assistants, Part 2: Three Operating Patterns That Actually Worked**  
   Turns the experimentation into practical operating models: a single-repo runbook, an Enterprise Copilot Fleet Controller, and a parent repository with scoped submodules. Emphasizes boundaries, auditability, critic loops, and avoiding over-broad context.

3. **Coding Assistants, Part 3: Human Expertise Is the Multiplier**  
   Explains where humans create leverage: vision, written communication, tooling, feedback loops, scope management, and integration knowledge. Positions AI-assisted delivery as a workflow shift that makes engineering fundamentals more valuable.
